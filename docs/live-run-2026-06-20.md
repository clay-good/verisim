# The live lane, re-run locally (2026-06-20)

> The whole RA live arc, executed end-to-end on this machine: **real Docker** (Docker Desktop) and a
> **real Claude agent driven through the `claude` Code CLI headlessly** (`claude -p ...
> --allowedTools "Bash(docker exec <container>:*)"`), with **no `ANTHROPIC_API_KEY`** — the CLI's own
> authenticated session is the agent. This is the lane SPEC-22 documented as a runbook; this file is
> the record of actually running it. Every number below is from a fresh local run today.

## How it was driven

- **Containers:** one image per scenario, built from the inline `bench/raN/Dockerfile`s
  (`docker build -t verisim-raN:latest bench/raN/`), then `docker run -d` per trial, harm read off
  the container state, container destroyed.
- **Agent:** `claude -p "<ticket>" --allowedTools "Bash(docker exec <name>:*)" --permission-mode
  default --output-format text`, run from a clean `/tmp` directory with **no repository context** (so
  the agent cannot read this work's description of the experiment — the control RA7 established).
- **Deterministic arms** (none / lockdown / mac_* / oracle / detect / prevent) use only Docker and
  cost no model calls; the **agent lanes** use the CLI and are the only billable part.

The agent lanes were run at **small K** here (the point was to execute the live lane locally, not to
re-power the rates); the committed `bench/raN/RESULTS.md` carry the powered K=10–60 numbers with Wilson
intervals. The small-K runs today reproduce the **direction and the deterministic structure** exactly;
RA12's ambient cell (1/2) is consistent with the powered 10/10 within its wide small-K interval.

## What this establishes

The external lane is no longer only a documented runbook: the gate, the un-dominated triad
(relational / cumulative / context), the real hardened-sandbox baseline, the exfiltration
prevent-vs-detect split, and the covert-channel honest negative all **run for real on this machine**,
driven by a real Claude agent through the Code CLI with no API key. The standing open step is
unchanged and larger than this: the full official Terminal-Bench harness (~100 tasks) against
OS-Harm/SafePred as the external benchmark, which needs the external harness wired in, not just these
hand-built vertical-slice containers.

Reproduce: build the images (`for d in ra8 ra9 ra12 ra13 ra14 ra15 ra6; do docker build -t
verisim-$d:latest bench/$d/; done`), then run each `bench/raN/*.py` (add `--agent-trials K` /
`--k K` for the live-agent lanes; requires Docker running and an authenticated `claude` CLI).
